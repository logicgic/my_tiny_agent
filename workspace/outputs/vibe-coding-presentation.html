什么是 Vibe Coding？

Vibe Coding 是一种直觉驱动的编程方式，强调：

  • 心流状态 — 完全沉浸，时间感消失
  • AI 协作 — 与 AI 结对编程，而非对抗
  • 快速迭代 — 先跑起来，再优化
  • 减少摩擦 — 让工具服务于思维

💡 核心理念

"不要与代码搏斗，要与代码共舞。让 AI 处理繁琐，你专注于创造。"

最佳 实践

✅ 应该做的

  • 小步提交 — 频繁 git commit
  • 写测试 — 让 AI 帮你生成
  • 代码审查 — 即使是 AI 写的
  • 记录决策 — 为什么这样实现
  • 保持好奇 — 理解每行代码

⚠️ 避免的陷阱

  • 盲目复制 — 不理解就粘贴
  • 过度依赖 — 失去独立思考
  • 忽略安全 — AI 可能生成漏洞
  • 不写注释 — 未来你会后悔
  • 跳过测试 — "看起来没问题"

第四章

常见 陷阱详解

🚨 陷阱 1: AI 幻觉

AI 可能生成不存在的 API 或错误的代码。始终验证关键代码，尤其是：

  • 第三方库调用 — 检查文档确认 API 存在
  • 安全相关代码 — 认证、加密、输入验证
  • 性能关键路径 — 可能有无意识的低效操作

🚨 陷阱 2: 上下文丢失

AI 不了解你的完整项目结构。解决方案：

  • 提供相关文件 — 把依赖的代码一起给 AI
  • 描述架构 — 简要说明项目结构
  • 使用 Cursor 等工具 — 它们能读取整个项目
